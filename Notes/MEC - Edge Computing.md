[Edge](https://www.sdxcentral.com/edge/definitions/whats-the-difference-between-edge-computing-and-mec/) computing refers to computing happening at the edge of a network. Various access points define the network edge, hence the name for the [edge standard](https://www.sdxcentral.com/edge/definitions/mobile-edge-computing-vs-multi-access-edge-computing/mec-standards/) architecture, Multi-access Edge Computing (MEC). Edge access points include cell phone towers, routers, WiFi, and local [data centers](https://www.sdxcentral.com/data-center/definitions/software-defined-data-center/). A secure edge includes encryption, system visibility, monitoring, intrusion detection and prevention, and access control.

A distributed [cloud](https://www.sdxcentral.com/cloud/definitions/what-is-cloud/) refers to having computation, storage, and networking in a micro-cloud located outside the centralized cloud. Examples of a distributed cloud include both [fog computing and edge computing](https://www.sdxcentral.com/edge/definitions/mobile-edge-computing-vs-multi-access-edge-computing/whats-difference-mec-fog-computing/). Establishing a distributed cloud situates computing closer to the end user, providing decreased latency and opportunities for increased security. Potential security solutions include a [blockchain-based security architecture](https://ieeexplore.ieee.org/document/8367295) or random auditing of data to [check for integrity](https://arxiv.org/abs/1805.02970). A distributed cloud also processes data in real-time.
